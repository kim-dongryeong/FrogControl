C# (.NET) native port (2026-07-06)
Added csharp/ - a faithful C# / .NET 8 (WinForms) rewrite of the whole script as a native
Windows tray application. Uses global low-level keyboard/mouse hooks (WH_KEYBOARD_LL /
WH_MOUSE_LL) so CapsLock-as-prefix and the tap-to-speed-up features work exactly like the AHK
build, with no AutoHotkey runtime. Every hotkey in shortcut list-en.txt is implemented; the
Show_Windows arrange engine, the two modal modes (Date/Time input, Mouse control), window/mouse
bookmarks, rotation, seek, snap/grid move and resize are all ported. Behaviour forks are exposed
as switches in frogcontrol.settings.json (MouseMoveBackend, ActivationBackend, SuppressCapsLock,
CapsLockTapToggles, EnableUnstableRotation) so they can be tested without recompiling. Build/run
with csharp/build.ps1; standalone .exe with csharp/publish.ps1. See csharp/README.md for details.
The AutoHotkey build (FrogControl.ahk) is unchanged and remains authoritative.

2.1.0 (2026-07-05)
Deduplication milestone: the whole refactor backlog is done. FrogControl.ahk now holds only
hotkeys and the auto-execute section (~2,600 lines); all functions live in lib/ modules
(FC_Bookmarks, FC_AltTab, FC_Arrange, FC_Helpers).
- removed +F5/+F6 (test aliases that became identical to +F1/+F2) and two debug hotkeys;
  shortcut lists now also document Win+Ctrl+Shift+Alt+RightClick and CapsLock+Tab
- four window-rotation hotkey bodies unified into WindowRotate_OnWheel()
- windowSeekArrow's four ~46-line direction blocks unified into one dot/cross-scored finder
  (windows straight above/below/beside are now always reachable; selection near diagonal
  boundaries can differ slightly from the old cone tests)
- twelve monitor-edge scan loops unified into Monitor_EdgeScan()
- 23 "AlwaysOnTop On/Off" pairs unified into PulseTop(); spread layouts activate one window
  per arrangement instead of every window in the loop (less flicker and focus churn)
- Show_Windows grid is computed from the window count and screen aspect ratio
  (replaces the fixed 800x450 cells and the grid-growing loop)
- DWM border compensation extended to edge moves (CapsLock+Shift+Arrow), edge resizes
  (Win+Alt+Shift+Arrow) and all spread layouts - windows sit flush everywhere now
- date-and-time input mode falls back to the mouse position in apps that expose no caret
- Ctrl+Shift+Alt+RightClick guards against an empty window under the mouse

2.0.0 (2026-07-05)
New version scheme. The 1.15 alpha line ends with 1.15.alpha.3 (2024); the long gap and the
audit + refactor overhauls below mark the jump to 2.x. From now on, plain MAJOR.MINOR.PATCH
with no alpha/beta suffixes: fixes bump the patch (2.0.1, 2.0.2, ...), bigger feature batches
bump the minor (2.1.0). The two 2026-07-05 releases below were renumbered retroactively into
the 2.0 line (they were first committed as 1.15.alpha.4 / 1.15.alpha.5).
- added a Reload item to the tray menu to apply script changes without exiting
  (re-running FrogControl.ahk also replaces the running instance thanks to #SingleInstance Force)

2.0.0-alpha.2 (2026-07-05, git commit ccc170b; originally numbered 1.15.alpha.5)
Refactoring and DWM border fix. The script file is now just "FrogControl.ahk" (the version lives in AppVersion and git, not in the filename).
- NEW: window snapping and grid placement compensate for the invisible DWM borders of Windows 10/11, so snapped windows sit flush with no ~7-14 px gaps (CapsLock+LButton snap zones, CapsLock+Ctrl+Arrow grid, Show_Windows grid)
- +F1/+F2 now delegate to Show_Windows() like +F5/+F6 (removed the old Explorer-only duplicate layout engine, ~830 lines; window filtering is now by process "Explorer.exe" instead of window class, same as +F5/+F6)
- extracted shared helpers: CollectMonitorInfo(), CountRecentTaps(), Volume_Adjust(), ShowHelp(), WheelScroll_CapsFast/OfficeFast(), DWM_WinMove()
- help window (En/Ko) is built by one function instead of two near-identical blocks
- added #Requires AutoHotkey v1.1 (clear error instead of confusing failures under AutoHotkey v2)
- Ctrl+Shift+Alt+RightClick (close window) can no longer hang forever (0.5 s activation timeout)
- ruler mode: BlockInput is now always released (10 s timeout) - a missed Space press can no longer leave input blocked
- Excel/Word COM scrolling wrapped in try/catch with plain-wheel fallback (no more error dialogs when Office is busy)
- grid division-by-zero guard for very small screens (RDP, portrait monitors)
- Show_Windows: safe early return when no matching window exists; stale monitor-index fix
- removed the broken, unused ObjectClone()
- net -871 lines (5,280 -> 4,409)

2.0.0-alpha.1 (2026-07-05, git commit 1a1a233; originally numbered 1.15.alpha.4)
Full audit pass: many bug fixes and performance optimizations. No hotkey changes.
Performance:
- CapsLock + Wheel scroll no longer enumerates every open window on each wheel notch (the biggest win)
- polling loops (window rotation/seek/move/resize, click-drag move/resize, ruler) no longer busy-wait; they sleep ~10 ms per pass, so a held gesture no longer pegs a CPU core
- monitor info (SysGet) is no longer re-queried on every iteration of the CapsLock+Arrow move loop
- windowSeekArrow recomputes window geometry only while an arrow key is actually pressed
- AltTab_List_SameType no longer enumerates all windows three times
- added #NoEnv, ListLines Off, #SingleInstance Force
Fixed:
- CapsLock modes (date input, mouse coordinates, mouse control) restored the INVERSE of the initial CapsLock state, flipping the CapsLock LED on every use
- keyboard auto-repeat flooded the modifier time-stamp ring, pushing every "press N times to speed up" feature straight to maximum
- monitor edge move/resize min/max scans ignored middle monitors (wrong edges on 3+ monitor setups)
- windowSeekArrow could never reach a window directly above/below (division by zero silently produced a blank)
- leftover debug code in the 3-window layout of Show_Windows (+F3/+F5): six debug windows, a blocking MsgBox and width corruption; the layout logic itself was repaired (sorting re-enabled, normalized widths, fresh sizes after WinRestore)
- copy-paste X/Y mix-up placed the tallest window at a wrong vertical position in the 3-explorer vertical layout (+F1 and Show_Windows)
- wheel window-rotation dropped queued scroll clicks; the stacking and popup rotation modes could corrupt each other's shared state
- Shift-only wheel scroll in Excel/Word wrongly turned CapsLock off afterwards
- Ctrl+Shift+RightClick (close tab) could hang forever waiting for activation, leaving the system beep disabled; each beep-toggling feature now keeps its own saved state
- window move/resize loops now act on a snapshot of the active window, so a focus change mid-gesture can no longer move/resize the wrong window
- grid move (CapsLock+Ctrl+Arrow) now detects the correct monitor for maximized windows (falls back to the primary if none matches)
- GUI leaks: help windows, mouse-bookmark markers and window-list views accumulated invisibly when reopened
- window transparency (Win+Alt+[ / ]) is clamped to the valid 0-255 range
- help window shows a notice instead of an empty list when a shortcut list file is missing
- Shift:: now passes the physical key through (~Shift::) instead of swallowing it and re-sending a synthetic tap
- removed a leftover debug combo (~Right & CapsLock::) that made CapsLock completely dead while the Right arrow was held
- Xbox / Movies & TV / Photos filtering in the window list actually compares the window class now
- misc: typo, 64-bit overflow in a sentinel constant, wrong monitor index recorded in the +F2 grid path
